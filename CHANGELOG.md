# 更新日志

本项目的所有重要更改均会记录在此文件中。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，
版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

---

## [1.1.3] - 2026-04-06

### 改进

- 💡 **错误信息增强** — 数据获取失败时增加针对性诊断建议（安装 curl_cffi、检查服务器地域、频率限制提示），替代原有笼统提示
- 🎯 **错误路径进度提示优化** — 修复数据获取失败后进度条显示「📝 正在生成最终报告」的误导问题，改为「📋 正在生成错误报告」

---

## [1.1.2] - 2026-04-06

### 修复

- 🐛 **数据获取失败后误报成功** — 修复 A 股/港股/美股数据获取全部失败（网络中断、连接断开、频率限制等）时，仍显示"✅ 数据预取完成"并继续执行分析的问题（#4）
- 🐛 **数据验证对长文本错误失效** — 修复 `_check_data_valid` 仅在数据长度 < 200 时检测失败关键词，导致多行 Markdown 格式的错误消息被误判为有效数据的问题
- 🐛 **分析流程未能中断** — 修复数据预取失败后仍无条件执行全部分析师节点，新增条件路由，数据获取失败时直接终止并输出错误提示
- 🐛 **进度回调误报** — 修复进度回调依赖特定字符串匹配（"数据源不齐全"），改为检查 error 字段是否存在，确保所有错误类型都能正确报告

---

## [1.1.1] - 2026-04-04

### 新增

- ⚡ **快速分析指令** — 新增 `/快速分析` 命令，跳过多空辩论环节，直接生成分析报告，大幅缩短等待时间
- 🎯 **快速分析模式架构** — 基于 LangGraph 条件边（conditional edges）实现，分析师生成报告后直接进入风险评估节点

### 修复

- 🐛 **TXT 导出 Emoji 乱码** — 修复 TXT 文本报告导出时 Emoji 显示为"口口口"方块的问题，导出前自动将 Emoji 替换为中文标签

---

## [1.1.0] - 2026-04-02

### 新增

- ✨ **PDF 导出开关** — 新增 `export_pdf` 配置项（默认开启），关闭后报告以 Markdown 分模块逐段发送，适合移动端阅读、防止消息截断
- 📋 **报告分模块输出** — 关闭 PDF 导出时，报告按 `##` 标题自动拆分为 6 段（标题信息、市场技术面、基本面、新闻面、多空辩论、风险评估）逐段发送
- 🔤 **内嵌中文字体** — PDF 生成使用项目自带 `fonts/NotoSansSC-Regular.ttf` 和 `fonts/NotoSansSC-Bold.ttf`，不再依赖系统字体，Docker 部署无需额外安装字体

### 修复

- 🐛 **PDF 中文渲染方块问题** — 修复 CSS `@font-face` 中 `format("opentype")` 与 `.ttf`（TrueType）字体格式不匹配，解决 PDF 导出中文显示"口口口"方块的问题

### 变更

- 🖼️ 更新插件头像

---

## [1.0.0] - 2026-03-26

### 初始发布

- 🔍 **智能股票分析** — 输入股票代码或名称，生成完整的分析报告
- 🤖 **多智能体架构** — 基于 LangGraph 编排市场分析师、基本面分析师、新闻分析师
- ⚖️ **多空辩论机制** — 多方研究员与空方研究员进行投资辩论，碰撞观点
- 🛡️ **风险评估裁判** — 风险裁判综合评估整体投资风险等级
- 🌐 **多市场支持** — A 股（沪深）、港股、美股全覆盖
- 📊 **多数据源** — akshare（A 股/港股/美股实时行情）+ yfinance（港股/美股基本面与新闻）
- 📄 **PDF + Markdown 双格式报告导出**
- 📱 **移动端支持** — 报告适配移动端查看
- 🔧 **OpenAI 兼容协议** — 支持智谱 GLM、DeepSeek、OpenAI、MiniMax 等多厂商模型
